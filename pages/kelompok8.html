Evaluasi Turunan & Integral Tak Tentu

Kelompok 8 Laju Perubahan Optimasi Titik Ekstrim Evaluasi Integral Tak Tentu Akumulasi

Turunan memiliki berbagai aplikasi nyata: mencari nilai ekstrem, menganalisis laju perubahan, hingga memecahkan masalah optimasi.


Integral tak tentu juga memiliki banyak aplikasi, terutama dalam menghitung luas daerah di bawah kurva dan dalam pemecahan masalah akumulasi.

8.A.1 Penyelesaian Masalah Terapan

Penyelesaian Masalah Terapan mencakup tiga sub-topik. Laju perubahan diselesaikan dengan mencari f'(x) lalu substitusi nilai x yang ditanyakan. Optimisasi diselesaikan dengan mencari titik kritis (f'= 0) lalu uji turunan kedua. Penentuan ekstrem memerlukan keduanya untuk membedakan titik maksimum dan minimum lokal.

  • Definisi Laju Perubahan

    • Laju perubahan adalah ukuran seberapa cepat suatu nilai berubah terhadap nilai lain. Sederhananya: jika sesuatu berubah, laju perubahan memberitahu kita seberapa besar perubahan itu terjadi setiap satuan waktu atau variabel.

    Laju Perubahan
    f(x) -> f'(x)

    keterangan:

    Laju perubahan fungsi f(x) pada titik tertentu dapat ditemukan dengan menghitung turunan pertama f'(x) pada titik tersebut.

  • Definisi Titik Ekstrem

    • Penentuan ekstrem adalah proses menemukan titik puncak dan titik lembah dari sebuah fungsi. Titik puncak disebut maksimum dan titik lembah disebut minimum.

    Syarat titik ekstrem
    f'(x) = 0

    keterangan:

    Untuk mencari titik ekstrem, kita perlu mencari nilai x dari turunan pertama sama dengan nol. Ini menunjukkan bahwa fungsi mungkin memiliki nilai maksimum atau minimum di titik tersebut.

  • Definisi Optimisasi

    • Optimisasi adalah proses mencari nilai maksimum atau minimum dari suatu fungsi. Dalam konteks laju perubahan, kita dapat menggunakan turunan pertama untuk menentukan titik-titik kritis yang dapat menjadi kandidat untuk nilai maksimum atau minimum.

    Syarat Optimasi Maksimum
    f''(x) < 0

    Keterangan

    Nilai Maksimum harus memiliki nilai lebih kecil dari nol (0)

    Syarat Optimasi Minimum
    f''(x) > 0

    Keterangan

    Nilai Minimum harus memiliki nilai lebih besar dari nol (0)

    Contoh Soal

    Contoh soal penyelesaian masalah terapan dengan turunan

    Diketahui: Sebuah perusahaan startup teknologi memodelkan keuntungan bersih (dalam juta rupiah) terhadap jumlah fitur yang diluncurkan per kuartal dengan fungsi: P(x) = -x3 + 9x2 - 15x + 2 dengan x = jumlah fitur antara 1 hingga 7.

    Ditanya: Tentukan jumlah fitur yang menghasilkan keuntungan maksimum menggunakan:

    • (a) Metode turunan
    • (b) Metode numerik dengan tabel nilai

    Kemudian lakukan pengambilan keputusan matematis: evaluasi ketepatan metode, bandingkan hasilnya, dan rumuskan implikasinya terhadap sistem perangkat lunak.

    Penyelesaian:

    1. Metode Turunan

      2. Untuk mencari jumlah fitur yang menghasilkan keuntungan maksimum, kita perlu mencari turunan pertama dari fungsi P(x).

      $$P(x) = -x^3 + 9x^2 - 15x + 2$$ $$P'(x) = -3x^2 + 18x - 15$$

      lalu menentukan titik kritisnya

      $$P'(x) = 0$$ $$-3x^2 + 18x - 15 = 0$$

      setiap nilai dapat dibagi 3, maka hasilnya:

      $$x^2 - 6x + 5 = 0$$ $$(x - 5)(x - 1) = 0$$

      maka, didapatkanlah titik

      $$x = 5 \quad \text{atau} \quad x = 1$$

      Sekarang, kita perlu menentukan apakah titik-titik ini adalah maksimum atau minimum.

      Hitung turunan kedua P''(x):

      $$P''(x) = -6x + 18$$

      Evaluasi P''(x) di titik-titik kritis:

      $$P''(1) = -6(1) + 18 = 12 > 0 \quad \text{(minimum)}$$ $$P''(5) = -6(5) + 18 = -12 < 0 \quad \text{(maksimum)}$$

      jadi, x = 1 adalah titik minimum dan x = 5 adalah titik maksimum.

      Selanjutnya, kita perlu menentukan nilai maksimum dan minimum dari fungsi P(x) pada titik-titik tersebut.

      Hitung P(1) dan P(5):

      $$P(1) = -(1)^3 + 9(1)^2 - 15(1) + 2 = -1 + 9 - 15 + 2 = -5$$ $$P(5) = -(5)^3 + 9(5)^2 - 15(5) + 2 = -125 + 225 - 75 + 2 = 27$$

      Dan juga batas interval diujung kanan = 7

      $$P(7) = -(7)^3 + 9(7)^2 - 15(7) + 2 = -343 + 441 - 105 + 2 = -5$$

      Jadi, keuntungan maksimum yang akan didapatkan pada fitur berjumlah 5 adalah 27 juta Rp.

    2. Metode Numerik dengan Tabel

      3. Metode numerik dengan tabel dapat digunakan untuk mendekati nilai maksimum dan minimum dari fungsi P(x) dengan cara menghitung nilai P(x) pada beberapa titik dalam interval [1 hingga 7].

      XPerhitunganP(x) hasilStatus
      1P(1) = -(1)^3 + 9(1)^2 - 15(1) + 2 = -1 + 18 - 13-5Turun
      2P(2) = -(2)^3 + 9(2)^2 - 15(2) + 2 = -8 + 36 - 30 + 20Naik
      3P(3) = -(3)^3 + 9(3)^2 - 15(3) + 2 = -27 + 81 - 45 + 211Naik
      4P(4) = -(4)^3 + 9(4)^2 - 15(4) + 2 = -64 + 144 - 60 + 222Naik
      5P(5) = -(5)^3 + 9(5)^2 - 15(5) + 2 = -125 + 225 - 75 + 227Maksimum
      6P(6) = -(6)^3 + 9(6)^2 - 15(6) + 2 = -216 + 324 - 90 + 220Turun
      7P(7) = -(7)^3 + 9(7)^2 - 15(7) + 2 = -343 + 441 - 105 + 2-5Turun

      jadi, pada metode numerik dengan tabel, kita dapat melihat bahwa nilai maksimum P(x) terjadi pada x = 5 dengan P(5) = 27 menyatakan bahwa hasilnya sama dengan metode turunan.

    Evaluasi Metode Penyelesaian

    • Metode turunan memberikan hasil yang lebih cepat dan langsung untuk menemukan titik maksimum. hasil didapatkan dalam 4 langkah. Turunan memberikan solusi eksak sekaligus membuktikan bahwa titik x = 5 adalah maksimum melalui uji turunan kedua P''(x)

    • Metode numerik dengan tabel memberikan pendekatan yang lebih intuitif dan mudah dipahami, tetapi kurang akurat dibandingkan metode turunan jika intervalnya terlalu besar. Numerik hanya menunjukkan P(5) lebih besar dari titik-titik yang diuji. Tanpa cukup titik, bisa saja melewatkan titik kritis lain jika fungsi lebih kompleks.

    Pengambilan Keputusan Matematis

    1. Implikasi terhadap pemodelan sistem perangkat lunak
      • Perencanaan roadmap: Tim produk sebaiknya merilis 5 fitur per kuartal. Lebih dari itu justru menurunkan keuntungan akibat kompleksitas, biaya maintenance, dan penurunan fokus pengguna.
      • Risiko over-engineering: Model menunjukkan P(7) = -5 (rugi). Meluncurkan 7 fitur sekaligus berpotensi merugikan perusahaan — validasi matematis ini mendukung keputusan bisnis yang lebih konservatif.

    8.B.1 Integral Tak Tentu

    Secara sederhana, integral adalah salah satu konsep utama dalam kalkulus yang merupakan kebalikan dari turunan (diferensial). Jika turunan berfungsi untuk mencari laju perubahan atau kemiringan suatu fungsi, maka integral berfungsi untuk mengembalikan fungsi tersebut ke bentuk aslinya.

  • Definisi Integral Tak Tentu

    • Integral tak tentu adalah proses untuk mencari fungsi asal (antiturunan) dari suatu fungsi yang telah diturunkan. Hasil dari integral tak tentu masih berupa fungsi dan selalu diakhiri dengan konstanta + C.

    Integral Tak Tentu
    ∫f(x) dx = F(x) + C

    keterangan:

    • ∫ -> Simbol integral, menunjukkan proses pengintegralan.
    • f(x) -> Fungsi integran, menunjukkan fungsi yang akan diintegralkan.
    • dx -> Deferensial/Turunan, menunjukkan perubahan kecil pada x.
    • F(x) -> Fungsi hasil integral, menunjukkan total hasil pengumpulan.
    • C -> Konstanta, menunjukkan bahwa ada nilai tetap yang ditambahkan pada hasil integral.

    Contoh Soal

    Contoh Soal Integral Tak Tentu

    Diketahui: Sebuah fungsi f(x) memiliki nilai f(x) = 9x2

    Ditanya: Tentukan bentuk integral tak tentu dari f(x)

    Penyelesaian:

    Gunakan Rumus:

    $$\int ax^n \, dx = \frac{a}{n+1} x^{n+1} + C$$
    • Ubah fungsi tersebut ke bentuk integral
      • $$\int f(x)dx = \int 9x^2 \, dx$$
    • Lalu kita subtitusikan nilai a diisi 9 dan n diisi oleh 2 ke dalam rumus
      • $$\int 9x^2 \, dx = \frac{9}{2+1} x^{2+1} + C$$
    • Maka didapatkanlah hasil yakni
      • $$\int f(x)dx = 3x^3 + C$$

      Jadi, integral tak tentu dari f(x) adalah 3x3 + C

    8.B.2 Hubungan antara integral dan turunan

  • Hubungan antara integral dan turunan

    • Hubungan antara integral dan turunan adalah salah satu penemuan paling penting dalam sejarah matematika. Hubungan ini disebut sebagai Hubungan Invers (Berkebalikan).

    Jika turunan adalah proses "memecah" atau mencari laju perubahan dari suatu fungsi, maka integral adalah proses "menggabungkan kembali" atau mengonstruksi ulang fungsi asal dari laju perubahannya. Karena sifat berkebalikan ini, integral sering disebut sebagai Antiturunan.

    Hubungan antara integral dan turunan

    Misalkan:

    $$ \begin{array}{ccc} & \xrightarrow{\text{Integral}} & \\ f(x) = 9x^2 & & F(x) = 3x^3 + C \\ & \xleftarrow{\text{Turunan}} & \end{array} $$

    keterangan:

    Jika f(x) = 9x2 kita integralkan, maka akan mendapatkan F(x) = 3x3 + C. begitu juga sebaliknya jika kita turunkan F(x) = 3x3 + C, maka akan kembali menjadi f(x) = 9x2.

    Ini menggambarkan sifat dasar dari integral dan turunan sebagai operasi yang saling membalikkan.

    8.B.3 Interpretasi Konstanta Integrasi (C)

  • Definisi Konstanta (C)

    • Konstanta integrasi (C) muncul dalam integral tak tentu karena adanya ketidakpastian dalam proses pengintegralan. Ketika kita mencari antiturunan dari suatu fungsi, kita sebenarnya mencari semua fungsi yang memiliki turunan yang sama dengan fungsi tersebut. Oleh karena itu, kita menambahkan konstanta C untuk mencakup semua kemungkinan fungsi yang berbeda.

    Konstanta (C)

    Misalkan:

    $$ \begin{array}{rcccl} f(x) = 3x^2 + 2 & \searrow & & & \\ \\ f(x) = 3x^2 + 10 & \longrightarrow & f'(x) = 9x^2 & \longrightarrow & F(x) = 3x^3 + C \\ \\ f(x) = 3x^2 + 100 & \nearrow & & & \end{array} $$

    keterangan:

    Setiap fungsi f(x) yang berbeda tidak peduli berapa besar nilainya pasti memiliki turunan yang sama, yaitu f'(x) = 9x2.

    Maka, antiturunan dari f'(x) adalah F(x) = 3x3 + C, di mana C mewakili setiap kemungkinan konstanta yang ditambahkan pada fungsi.

    8.B.4 Integral Tak Tentu dalam konsep akumulasi perubahan

  • Definisi Konsep Akumulasi Perubahan

    • Akumulasi perubahan adalah menghitung hasil total dari perubahan yang terjadi terus-menerus. Jika kita tahu seberapa cepat sesuatu bertambah atau berkurang setiap saat (laju perubahan), kita bisa menggunakan Integral Tak Tentu untuk menghitung berapa jumlah total yang terkumpul (akumulasi) dari waktu ke waktu.

    Contoh Soal

    Contoh Soal Integral Tak Tentu dalam Konsep Akumulasi Perubahan

    Soal:

    Diketahui: Sebuah perusahaan menyimpan data pengguna yang terus bertambah. Laju pertumbuhan data (dalam Gigabyte per hari) terhadap waktu t (dalam hari) dimodelkan oleh fungsi f(t) = 3t2 + 4t + 10. Jika pada hari ke-0 total data awal yang tersimpan adalah 100GB.

    Ditanya:

    • Tentukan bentuk fungsi akumulasi data secara umum!
    • Tentukan kapasitas data hari ke-5!

    Penyelesaian:

    1. Menentukan bentuk fungsi akumulasi data secara umum
      • Ubah fungsi tersebut ke bentuk integral
        • $$\int f(t)dt = \int (3t^2 + 4t + 10) \, dt$$
      • Lalu kita subtitusikan nilai a diisi 3, 4, 10 dan n diisi oleh 2, 1, 0 ke dalam rumus
        • $$\int (3t^2 + 4t + 10) \, dt = \left( \frac{3}{2+1} t^{2+1} + \frac{4}{1+1} t^{1+1} + \frac{10}{0+1} t^{0+1} \right) + C$$
      • maka, didapatkanlah f(t)
        • $$f(t) = t^3 + 2t^2 + 10t + C$$
      • Kemudian kita hitung nilai C dengan menggunakan kondisi awal yakni f(0) = 100
        • $$f(0) = 0^3 + 2(0)^2 + 10(0) + C$$ $$100 = C$$ $$\Longleftrightarrow$$ $$C = 100$$
      • lalu kita subtitusikan konstanta(C) = 100 ke dalam fungsi f(t)
        • $$f(t) = t^3 + 2t^2 + 10t + 100$$

        Maka, didapatkanlah Bentuk umum dari fungsi akumulasi data tersebut yakni

        f(t) = t 3 + 2t 2 + 10t + 100

    2. Menentukan kapasitas data hari ke-5
      • Substitusikan t = 5 ke dalam fungsi f(t)
        • $$f(5) = 5^3 + 2(5)^2 + 10(5) + 100$$
      • Hitung nilai f(5)
        • $$f(5) = 125 + 50 + 50 + 100$$
        $$f(5) = 325$$

        Jadi, kapasitas data hari ke-5 adalah 325GB

    Latihan Soal

    1. Soal 1

      [Evaluasi Turunan - Optimasi Kinerja Server]
      Sebuah sistem server memiliki kapasitas pemrosesan request API per milidetik yang dimodelkan dengan fungsi P(x) = -5x2 + 100x + 50, di mana x adalah jumlah thread CPU yang dialokasikan. Berapa banyak thread yang harus dialokasikan untuk mencapai jumlah pemrosesan maksimal?

      Lihat Jawaban

      Penyelesaian:

      Untuk mencari pemrosesan maksimal, kita mencari titik di mana turunan pertama P'(x) = 0.

      $$P'(x) = -10x + 100$$

      $$-10x + 100 = 0$$

      $$10x = 100$$

      $$x = 10$$

      Jadi, untuk mendapatkan kapasitas pemrosesan maksimal, sistem harus mengalokasikan 10 thread.

    2. Soal 2

      [Evaluasi Turunan - Efisiensi Biaya Cloud]
      Biaya operasional server cloud per jam bergantung pada jumlah instance n yang aktif, dengan fungsi biaya C(n) = 2n2 - 40n + 500 (dalam ribuan rupiah). Berapa jumlah instance yang harus diaktifkan agar biaya operasional minimum, dan berapakah biaya minimum tersebut?

      Lihat Jawaban

      Penyelesaian:

      Syarat biaya minimum adalah turunan pertama fungsi biaya sama dengan nol (C'(n) = 0).

      $$C'(n) = 4n - 40$$

      $$4n - 40 = 0$$

      $$4n = 40$$

      $$n = 10$$

      Substitusikan n = 10 ke fungsi awal:

      $$C(10) = 2(10)^2 - 40(10) + 500$$

      $$C(10) = 200 - 400 + 500 = 300$$

      Jadi, biaya akan minimum dengan mengaktifkan 10 instance dengan biaya sebesar 300 ribu rupiah.

    3. Soal 3

      [Evaluasi Turunan - Model Machine Learning]
      Dalam pelatihan sebuah model algoritma, fungsi tingkat kerugian (Loss Function) terhadap penyesuaian bobot w dinyatakan sebagai L(w) = w 4 - 8w 3 + 18w 2. Tentukan nilai bobot w yang membuat gradien (turunan pertama) bernilai nol (berpotensi menjadi titik minimum kerugian)!

      Lihat Jawaban

      Penyelesaian:

      Tentukan turunan pertama dan samakan dengan nol (L'(w) = 0).

      $$L'(w) = 4w^3 - 24w^2 + 36w$$

      $$4w^3 - 24w^2 + 36w = 0$$

      Faktorkan dengan mengeluarkan 4w:

      $$4w(w^2 - 6w + 9) = 0$$

      $$4w(w - 3)^2 = 0$$

      Maka diperoleh nilai w = 0 atau w = 3.

    4. Soal 4

      [Evaluasi Turunan - Laju Pertumbuhan Memori]
      Penggunaan memori dari sebuah algoritma pengurutan terhadap ukuran input x (dalam ribuan data) didefinisikan sebagai $$M(x) = \frac{1}{3}x^3 + 2x^2 + 5x$$ (dalam MB). Berapakah laju perubahan (pertumbuhan) memori seketika pada saat ukuran input x = 3?

      Lihat Jawaban

      Penyelesaian:

      Laju perubahan seketika adalah turunan pertama M'(x).

      $$M'(x) = x^2 + 4x + 5$$

      Substitusi x = 3 ke dalam turunan pertama:

      $$M'(3) = (3)^2 + 4(3) + 5$$

      $$M'(3) = 9 + 12 + 5 = 26$$

      Jadi, laju pertumbuhan memorinya adalah 26 MB per ribu data.

    5. Soal 5

      [Evaluasi Turunan - Fluktuasi Transfer Jaringan]
      Kecepatan transfer data pada sebuah router di waktu t detik dimodelkan dengan V(t) = t3 - 9t2 + 24t + 10 (dalam Mbps). Pada detik ke berapa kurva kecepatan mengalami titik belok (percepatan transfer sama dengan nol)?

      Lihat Jawaban

      Penyelesaian:

      Titik belok dicari dengan menyamakan turunan kedua dengan nol (V''(t) = 0).

      Turunan pertama (percepatan/perlambatan kecepatan):

      $$V'(t) = 3t^2 - 18t + 24$$

      Turunan kedua (perubahan percepatan):

      $$V''(t) = 6t - 18$$

      $$6t - 18 = 0$$

      $$6t = 18$$

      $$t = 3$$

      Jadi, titik belok transfer jaringan terjadi pada detik ke-3.

    6. Soal 6

      [Integral Tak Tentu - Fungsi Bug Fixing]
      Tim Quality Assurance (QA) mendeteksi bahwa laju perbaikan bug harian dalam sebuah proyek software dinyatakan dengan r(t) = 6t2 + 4t (dalam bug/hari). Tentukan fungsi umum akumulasi bug yang telah diperbaiki B(t) menggunakan integral tak tentu!

      Lihat Jawaban

      Penyelesaian:

      Fungsi total B(t) adalah antiturunan (integral tak tentu) dari r(t).

      $$B(t) = ∫ (6t² + 4t) dt$$

      $$B(t) = (6/3)t³ + (4/2)t² + C$$

      $$B(t) = 2t³ + 2t² + C$$

      Jadi, fungsi akumulasinya adalah B(t) = 2t³ + 2t² + C.

    7. Soal 7

      [Integral Tak Tentu - Akumulasi Baris Kode]
      Laju penulisan kode oleh seorang developer (dalam baris kode per jam) dinyatakan dengan persamaan P'(x) = 3x2 - 2x + 15. Jika pada jam ke-0 belum ada kode yang ditulis (P(0) = 0), tentukan fungsi P(x) yang menyatakan total akumulasi baris kode yang diselesaikan!

      Lihat Jawaban

      Penyelesaian:

      Cari fungsi integral tak tentu terlebih dahulu:

      $$P(x) = \int (3x^2 - 2x + 15) \, dx$$

      $$P(x) = x^3 - x^2 + 15x + C$$

      Gunakan syarat awal $$P(0) = 0$$ untuk mencari nilai konstanta C:

      $$P(0) = (0)^3 - (0)^2 + 15(0) + C = 0 \implies C = 0$$

      Jadi, fungsi akumulasi total kodenya adalah P(x) = x3 - x2 + 15x.

    8. Soal 8

      [Integral Tak Tentu - Akumulasi Request API]

      Diketahui: Sebuah server API menerima request yang terus bertambah saat jam sibuk. Laju masuknya request (dalam ribu request per jam) terhadap waktu t (dalam jam) dimodelkan oleh fungsi r(t) = 6t2 + 2t + 5. Pada jam ke-0 (saat server baru di-restart), sudah terdapat antrean sebanyak 50 ribu request yang belum diproses dari sesi sebelumnya.

      Ditanya:

      • Tentukan bentuk fungsi akumulasi request secara umum!
      • Tentukan total akumulasi request pada jam ke-4!

      Lihat Jawaban

      Penyelesaian:

      1. Menentukan bentuk fungsi akumulasi request secara umum
        • Ubah fungsi tersebut ke bentuk integral
          • $$R(t) = \int (6t^2 + 2t + 5) \, dt$$
        • Lalu kita subtitusikan nilai a diisi 6, 2, 5 dan n diisi oleh 2, 1, 0 ke dalam rumus
