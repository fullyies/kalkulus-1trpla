Kelompok 9 Definisi Integral

Integral adalah konsep dasar dalam kalkulus yang digunakan untuk menghitung luas, volume dan akumulasi suatu besaran.

9.1 Integral Tentu dan tak Tentu

Integral tentu adalah integral yang memiliki nilai batas atas dan batas bawah.

9.2 Pendekatan Kontinu dan Diskret

Dunia nyata bersifat kontinu (tak terhingga dan mulus), sedangkan sistem komputer bersifat Diskret (terbatas dan berbasis sample). Dalam pemograman, kita menggunakan metode Jumlah Riemann untuk memecahkan area kontinu menjadi potongan-potongan kecil agar bisa diproses oleh algoritma.

[ Kelompok 9: Definisi Pendekatan Kontinu dan Diskret ]

Contoh Soal

Contoh Soal 9.1 — Integral Tentu & Tak Tentu

Soal:

Diketahui sebuah fungsi f(x) = 4x. Tentukanlah:
a. Hasil dari integral tak tentu: ∫ 4x dx
b. Hasil dari integral tentu dengan batas x=1 sampai x=3: ₁∫³ 4x dx

Penyelesaian:

a. Menyelesaikan Integral Tak Tentu:
Gunakan rumus dasar integral ∫ xⁿ dx = (1 / n+1) xⁿ⁺¹ + C.
• ∫ 4x dx = (4 / 1+1) x¹⁺¹ + C
• Hasil = 2x² + C (berupa fungsi umum)

b. Menyelesaikan Integral Tentu:
Gunakan fungsi antiturunan F(x) = 2x² dari poin (a), lalu masukkan batasnya:
• Batas Atas (x = 3): F(3) = 2(3)² = 2(9) = 18
• Batas Bawah (x = 1): F(1) = 2(1)² = 2(1) = 2

Hasil Akhir (Batas Atas - Batas Bawah):
• F(3) - F(1) = 18 - 2 = 16 (berupa nilai/angka pasti)

Contoh Soal 9.2 — Pendekatan Kontinu dan Diskret

Soal:

Sebuah server cloud mencatat kecepatan transfer data (bandwidth) yang terus berubah setiap detiknya (kontinu). Berdasarkan grafik monitoring, fungsi kecepatan data tersebut diperkirakan mengikuti rumus f(t) = 3t2 + 2t (dalam Megabit per detik/Mbps). Sebagai DevOps Engineer di tim TRPL, Anda diminta menghitung perkiraan Total Akumulasi Data yang ditransfer dari detik ke-1 hingga detik ke-5 (t = 1 hingga t = 5). Karena sistem komputer bekerja secara diskret, hitunglah estimasi tersebut menggunakan metode Jumlah Riemann Kanan dengan membagi waktu menjadi 4 interval kotak yang sama (Δt = 1 detik).

Penyelesaian:

Langkah 1: Menentukan Lebar Kotak (Δt)
Diketahui batas bawah a = 1, batas atas b = 5, dan jumlah interval n = 4.
Δt = (b - a) / n = (5 - 1) / 4 = 1 detik

Langkah 2: Menentukan Titik Sampel Diskret (Riemann Kanan)
Karena menggunakan Riemann Kanan, kita mengambil ujung kanan dari setiap interval: 2, 3, 4, dan 5 detik.

Langkah 3: Menghitung Tinggi Kotak (Nilai Fungsi f(t))

  • Ujung 1 (t = 2): f(2) = 3(2)2 + 2(2) = 12 + 4 = 16 Mbps
  • Ujung 2 (t = 3): f(3) = 3(3)2 + 2(3) = 27 + 6 = 33 Mbps
  • Ujung 3 (t = 4): f(4) = 3(4)2 + 2(4) = 48 + 8 = 56 Mbps
  • Ujung 4 (t = 5): f(5) = 3(5)2 + 2(5) = 75 + 10 = 85 Mbps

Langkah 4: Menghitung Total Akumulasi (Jumlah Riemann)

Konsep: Total Akumulasi ≈ Σ (Tinggi Kotak × Lebar Kotak)
Total Akumulasi ≈ [f(2) × 1] + [f(3) × 1] + [f(4) × 1] + [f(5) × 1]
Total Akumulasi ≈ 16 + 33 + 56 + 85 = 190 Megabit

Jadi, estimasi total data diskret yang mengalir melalui server software tersebut adalah sebesar 190 Megabit.

Latihan Soal

Kelompok 9 — Silahkan mengisi latihan soal di bawah ini!.

  1. Soal 1: Integral Tak Tentu (Performa CPU)

    Sebuah algoritma optimasi mengalami perubahan laju penggunaan memori RAM (dalam MB/detik) yang dinyatakan oleh fungsi f'(t) = 6t2 - 4t + 5. Tentukan fungsi total penggunaan memori f(t) jika diketahui pada detik ke-0 (t = 0) memori awal yang digunakan sistem adalah 100 MB!

    Lihat Jawaban
    Penyelesaian:
    Cari integral tak tentu dari f'(t):
    f(t) = ∫ (6t2 - 4t + 5) dt
    f(t) = (6/3)t3 - (4/2)t2 + 5t + C
    f(t) = 2t3 - 2t2 + 5t + C

    Substitusi nilai awal t = 0, f(0) = 100:
    100 = 2(0)3 - 2(0)2 + 5(0) + C → C = 100

    Jawaban Akhir: f(t) = 2t3 - 2t2 + 5t + 100

  2. Soal 2: Integral Tak Tentu (Kecepatan Unduh Data)

    Laju unduh (*download speed*) sebuah aplikasi meningkat berdasarkan fungsi f'(t) = 3t1/2 + 2 (dalam Mbps). Tentukan rumus umum total ukuran file f(t) yang berhasil diunduh setelah t detik!

    Lihat Jawaban
    Penyelesaian:
    f(t) = ∫ (3t1/2 + 2) dt
    f(t) = (3 / (3/2)) * t3/2 + 2t + C
    f(t) = 2t3/2 + 2t + C

    Jawaban Akhir: f(t) = 2t√t + 2t + C

  3. Soal 3: Integral Tentu (Akumulasi Traffic Request API)

    Sebuah endpoint API menerima request dengan laju r(t) = 4t3 + 3t2 request per menit. Hitunglah total request yang masuk ke server dari menit ke-1 (t = 1) hingga menit ke-2 (t = 2)!

    Lihat Jawaban
    Penyelesaian:
    Hitung integral tentu dengan batas bawah 1 dan batas atas 2:
    ∫12 (4t3 + 3t2) dt = [t4 + t3]12

    Substitusi batas atas (t = 2): (24 + 23) = 16 + 8 = 24
    Substitusi batas bawah (t = 1): (14 + 13) = 1 + 1 = 2

    Total Request = 24 - 2 = 22
    Jawaban Akhir: 22 Request

  4. Soal 4: Integral Tentu (Konsumsi Daya IoT Device)

    Sensor IoT mengonsumsi daya listrik dengan laju P(t) = 9t2 - 2t + 4 miliwatt. Hitunglah total energi (dalam milijoule) yang dihabiskan sensor pada rentang waktu t = 0 hingga t = 3 detik!

    Lihat Jawaban
    Penyelesaian:
    ∫03 (9t2 - 2t + 4) dt = [3t3 - t2 + 4t]03

    Substitusi t = 3: 3(3)3 - (3)2 + 4(3) = 3(27) - 9 + 12 = 81 - 9 + 12 = 84
    Substitusi t = 0: 3(0) - 0 + 0 = 0

    Total Energi = 84 - 0 = 84
    Jawaban Akhir: 84 milijoule

  5. Soal 5: Penerapan Diskret (Jumlah Riemann Kiri - Bandwidth)

    Sebuah sistem monitoring jaringan mencatat kecepatan upload data server f(t) per detik seperti tabel berikut:
    t (detik): 0, 1, 2, 3, 4
    f(t) (Mbps): 10, 15, 20, 25, 30
    Estimasikan total data yang diupload dari t = 0 hingga t = 4 menggunakan Riemann Kiri dengan Δt = 1!

    Lihat Jawaban
    Penyelesaian:
    Karena menggunakan Riemann Kiri, ambil nilai f(t) dari titik kiri interval (t = 0, 1, 2, 3):
    Total ≈ Δt * [f(0) + f(1) + f(2) + f(3)]
    Total ≈ 1 * [10 + 15 + 20 + 25]
    Total ≈ 70 Megabit
    Jawaban Akhir: 70 Megabit

  6. Soal 6: Penerapan Diskret (Jumlah Riemann Kanan - Log Analisis)

    Berdasarkan log aktivitas web, laju penambahan pengguna baru per jam mengikuti pola f(t) = 2t + 1. Hitunglah estimasi total pengguna baru dari jam ke-0 hingga jam ke-3 (t = 0 sampai t = 3) menggunakan metode Riemann Kanan dengan membaginya menjadi 3 interval (Δt = 1)!

    Lihat Jawaban
    Penyelesaian:
    Batas: t=0 hingga t=3 dengan n=3, maka Δt = 1. Titik kanan interval adalah t = 1, 2, 3.
    f(1) = 2(1) + 1 = 3
    f(2) = 2(2) + 1 = 5
    f(3) = 2(3) + 1 = 7

    Total ≈ 1 * [f(1) + f(2) + f(3)] = 1 * [3 + 5 + 7] = 15
    Jawaban Akhir: 15 pengguna baru

  7. Soal 7: Integral Tak Tentu (Skalabilitas Server Node.js)

    Pertumbuhan beban antrean *task* pada server Node.js dinyatakan oleh f'(t) = 12t3 - 6t. Tentukan fungsi f(t) yang merepresentasikan jumlah *task* dalam antrean pada waktu t jika pada t = 1 jumlah antrean adalah 5 *task*!

    Lihat Jawaban
    Penyelesaian:
    f(t) = ∫ (12t3 - 6t) dt = 3t4 - 3t2 + C
    Substitusi f(1) = 5:
    5 = 3(1)4 - 3(1)2 + C
    5 = 3 - 3 + C → C = 5

    Jawaban Akhir: f(t) = 3t4 - 3t2 + 5

  8. Soal 8: Integral Tentu (Pemrosesan Data Gambar/Citra)

    Dalam algoritma computer vision, gradien intensitas piksel horizontal pada sebuah baris gambar dinyatakan oleh f(x) = 6x2 - 2x. Hitung nilai akumulasi perubahan intensitas dari koordinat x = 1 hingga x = 4!

    Lihat Jawaban
    Penyelesaian:
    ∫14 (6x2 - 2x) dx = [2x3 - x2]14

    Substitusi x = 4: 2(4)3 - (4)2 = 2(64) - 16 = 128 - 16 = 112
    Substitusi x = 1: 2(1)3 - (1)2 = 2 - 1 = 1

    Total Akumulasi Intensitas = 112 - 1 = 111
    Jawaban Akhir: 111

  9. Soal 9: Penerapan Diskret (Jumlah Riemann Tengah - Pengukuran Latency)

    Sebuah pengujian sistem mengukur fluktuasi latency f(t) = t2 (dalam milidetik). Estimasikan total penumpukan latency dari t = 0 hingga t = 2 detik menggunakan 2 interval (n = 2) dengan metode Riemann Tengah (Midpoint)!

    Lihat Jawaban
    Penyelesaian:
    Interval pertama [0, 1] → titik tengah t = 0.5
    Interval kedua [1, 2] → titik tengah t = 1.5
    Lebar interval Δt = (2-0)/2 = 1

    f(0.5) = (0.5)2 = 0.25
    f(1.5) = (1.5)2 = 2.25

    Total ≈ 1 * [f(0.5) + f(1.5)] = 1 * [0.25 + 2.25] = 2.5
    Jawaban Akhir: 2.5 milidetik

  10. Soal 10: Penerapan Diskret vs Kontinu (Error Batas Atas)

    Aplikasi web scraping mencatat rate limit request f(t) = 4t. Jika Anda menghitung area dari t = 0 hingga t = 2 dengan 2 interval (n = 2) menggunakan Riemann Kanan, berapakah selisih (error) hasil diskret tersebut jika dibandingkan dengan nilai integral tentu aslinya (kontinu)?

    Lihat Jawaban
    Penyelesaian:
    1. Secara Kontinu (Integral Tentu):
    ∫02 (4t) dt = [2t2]02 = 2(2)2 - 0 = 8

    2. Secara Diskret (Riemann Kanan, Δt = 1, titik kanan t = 1 dan 2):
    Total ≈ 1 * [f(1) + f(2)] = 1 * [4(1) + 4(2)] = 4 + 8 = 12

    3. Menghitung Selisih (Error):
    Selisih = Nilai Diskret - Nilai Kontinu = 12 - 8 = 4
    Jawaban Akhir: 4
